# Adam's Awesome Tools - Verge3D Blockly Extension PRD

## Overview
Adam's Awesome Tools is a custom Verge3D Blockly extension that enhances the puzzle creation experience with advanced block manipulation capabilities. The extension provides intuitive keyboard shortcuts for moving blocks within chains, making puzzle development more efficient and user-friendly. The project targets Verge3D developers and puzzle creators who need enhanced block management tools.

## Core Features

### 1. Custom Block Types
- **Label Field Block**: A simple dummy block that adds visual labels and console logging functionality
- **Step Block**: Advanced block with keyboard-based movement capabilities
- **Custom Category**: "Adam's Custom Tools" category with distinctive orange color (#de3900)

### 2. Block Movement System
- **Keyboard Shortcuts**: 'u' key moves blocks up in chain, 'j' key moves blocks down
- **Smart Movement Logic**: Prevents invalid moves that would break block structure
- **Stack Preservation**: Maintains visual positioning and layout during moves
- **Local Stack Support**: Handles movement within statement inputs (inner stacks)

### 3. Plugin Integration
- **Verge3D Integration**: Seamless integration with Verge3D puzzle system
- **Blockly v8 Compatibility**: Built for the latest Blockly version
- **Category Organization**: Custom category for easy discovery and use

## Technical Architecture

### System Components
- **Block Definitions**: XML-based block templates with custom JavaScript
- **Plugin Initialization**: init.plug file registers blocks and categories
- **Movement Engine**: JavaScript-based block manipulation system
- **Event Handling**: Keyboard event listeners for movement shortcuts

### Data Models
- **Block Structure**: Hierarchical block connections (previous/next/parent)
- **Input Types**: Support for statement inputs and dummy blocks
- **Position Tracking**: Relative positioning system for visual consistency

### APIs and Integrations
- **Blockly API**: Direct integration with Blockly's block manipulation methods
- **Verge3D Puzzle System**: Plugin registration and category management
- **DOM Events**: Keyboard event handling for user interactions

### Infrastructure Requirements
- **File Structure**: Organized block definitions and JavaScript files
- **Plugin Loading**: Verge3D plugin system integration
- **Browser Compatibility**: Modern browser support for Blockly v8

## Risks and Mitigations

### Technical Challenges
- **Blockly API Changes**: Risk of breaking changes in Blockly updates
  - *Mitigation*: Version pinning and compatibility testing
- **Complex Block Structures**: Risk of breaking nested block relationships
  - *Mitigation*: Comprehensive validation and testing
- **Performance Issues**: Risk of slowdown with large block chains
  - *Mitigation*: Efficient algorithms and optimization

### MVP Scope Management
- **Feature Creep**: Risk of adding too many features too quickly
  - *Mitigation*: Focus on core movement functionality first
- **User Adoption**: Risk of low adoption due to complexity
  - *Mitigation*: Simple, intuitive keyboard shortcuts
- **Integration Issues**: Risk of conflicts with other Verge3D plugins
  - *Mitigation*: Proper namespacing and isolation

### Resource Constraints
- **Development Time**: Risk of scope exceeding available time
  - *Mitigation*: Prioritize core functionality over nice-to-have features
- **Testing Complexity**: Risk of insufficient testing coverage
  - *Mitigation*: Automated testing for critical movement functions
